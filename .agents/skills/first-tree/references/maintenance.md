# Maintenance

## Before Work

- inspect the current root
- read the relevant tree nodes
- identify whether the task changes decisions or only implementation

## During Work

- decide in the tree, execute in source systems
- keep tree changes minimal and durable
- prefer explicit links between tree PRs and code PRs

## After Work

- ask whether the tree needs updating
- if the task changed shared decisions or constraints, update the tree first
- verify tree structure before merge when tree files changed

## Review Discipline

- owner review matters more than broad visibility
- do not merge contradictory tree and code changes separately without a clear linking story
