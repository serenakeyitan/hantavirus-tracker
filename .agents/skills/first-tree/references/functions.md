# Tree Functions

Use First Tree for information that would otherwise require reading many repos,
PRs, or conversations to reconstruct.

## Put These In The Tree

- durable decisions
- constraints that shape implementation choices
- ownership and review routing
- cross-repo relationships
- rationale future agents will need before acting

## Keep These Out Of The Tree

- API signatures
- code-level implementation detail
- ephemeral task chatter
- duplicated README material that already lives with the code

The tree should compress expensive context, not mirror the source repo.
